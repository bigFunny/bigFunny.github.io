bean作用范围

范围 描述
singleton(单例模式) 每次getbean都是同个对象
prototype(原型模式) 每次getbean都是不同对象
request 将单个 bean 定义限定为单个 HTTP 请求的生命周期。也就是说，每个 HTTP 请求都有自己的 bean 实例，该实例是在单个 bean 定义的后面创建的。仅在 Web 感知 Spring 的上下文中有效ApplicationContext。
session 将单个 bean 定义限定为 HTTP 的生命周期Session。仅在 Web 感知 Spring 的上下文中有效ApplicationContext。
application 将单个 bean 定义限定为ServletContext. 仅在 Web 感知 Spring 的上下文中有效ApplicationContext。
websocket 将单个 bean 定义限定为WebSocket. 仅在 Web 感知 Spring 的上下文中有效ApplicationContext。

AOP

静态代理

  • 在增加一个新的代理类来代理要代理的对象，比如婚介所，中介等
  • 代理是在原类的基本上增加功能，比如中介会帮你介绍房子，看房，而原类只愿意租房的，懒得帮购房者看。

动态代理

  • 动态代理是基于反射实现，所以他的创建类也是动态创建
  • 比如房东突然不懒了，增加帮看房功能，房东这个类就变了
  • 动态代理实现方式
    • 基于接口实现：JDK动态代理
    • 基于类来实现：cglib
    • java字节码实现：javasist

SpringAOP实现动态代理

AOP相关概念

1、AOP的优点

  • 降低模块之间的耦合度
  • 使系统更容易扩展
  • 更好的代码复用
  • 非业务代码更加集中，不分散，便于统一管理
  • 业务代码更加简洁纯粹，不掺杂其他的代码的影响

2、AOP中出现的一些概念

  • 切面Aspect：横切关注点，被模块化的抽象对象
  • 通知Advice：切面对象完成的工作（非业务代码）
  • 目标Target：被通知的对象（即被横切的对象）
  • 连接点joinpoint：通知要插入业务代码的具体位置（如Spring实现中的JoinPoint）
  • 切点Pointcut：AOP通过切点定位到连接点
  • 织入Weave：指将aspects应用到target对象并导致proxy对象创建的过程称为织入。
  • 引入Introduction：在不修改类代码的前提下，Introduction可以在运行期为类(代理类)动态地添加一些方法或Field。

3、AOP中的通知

  • 前置通知
  • 后置通知
  • 最终通知
  • 环绕通知
  • 异常通知
